# ReadMeCraft 修正方案 v1

基于 [v1 审计报告](v1.md) 提出的具体修正方案。按优先级排列。

## P0 — 三、AI 输出质量

### 3.1 README 质量评分

**问题：** validate-retry 只检查结构性缺失（表格行数、mermaid 类型），不评估语义质量。

**方案：** 在生成结果返回前端后，前端调用一次轻量评分请求。

```
POST /api/score-readme
{ "markdown": "...", "repoUrl": "..." }
→ { "score": 85, "checks": { "sections": true, "substance": true, "details": false } }
```

评分维度：
- 章节完整性：是否包含 README 必备章节（Installation、Usage、API 等），根据项目类型加权
- 内容充实度：段落是否过于简短（每个章节至少 2-3 句），是否有空洞占位
- 技术细节数：代码示例、配置项、命令行的数量
- 中文质量：是否混入未翻译的英文占位

**涉及文件：**
- 新增 `server/src/services/readme-scorer.ts` — 评分逻辑（用 MiniMax 做一次轻量分析）
- 新增 `server/src/routes/score.ts` — `/api/score-readme` 路由
- 修改 `server/src/routes/generate.ts` — 生成完成后可自动调评分
- 修改 `frontend/src/components/GenerateSection.tsx` — 在结果卡上显示评分（绿/黄/红仪表盘）
- 修改 `frontend/src/context/AppContext.tsx` — 新增 `qualityScore` state

**范围：** 中。新增约 150 行后端逻辑 + 50 行前端展示。

### 3.2 反馈重生成预设按钮

**问题：** 反馈重生成需要用户手动写 prompt，门槛高。

**方案：** FeedbackCard 增加 4 个预设修正按钮，点击后自动填充 textarea：

| 按钮 | 对应的修正 prompt |
|------|------------------|
| 内容太简略 | 内容过于简略，请扩充每个章节的细节，增加具体的技术描述和使用示例 |
| 技术栈描述不准确 | 技术栈描述不准确，请根据仓库代码中的实际依赖修正框架和工具列表 |
| 安装步骤有误 | 安装步骤可能需要补充环境要求、前置依赖和配置说明 |
| 需要更多使用示例 | 请增加更多代码示例，覆盖常见使用场景和高级用法 |

**涉及文件：**
- 修改 `frontend/src/components/FeedbackCard.tsx` — 按钮组 + 点击填充逻辑

**范围：** 小。约 20 行改动。

### 3.3 严格模式开关

**问题：** 用户无法控制 AI 的"发挥程度"。

**方案：** 在生成页面的模板选择区下方加一个 toggle 开关"严格模式"。

- 严格模式开启时：buildSystemPrompt 追加约束，强制骨架结构、最少段落长度、禁止占位符
- 严格模式关闭时：现有的宽松 prompt，AI 自由发挥

**涉及文件：**
- 修改 `frontend/src/context/AppContext.tsx` — 新增 `strictMode` state
- 修改 `frontend/src/components/TemplateSelector.tsx` — 加 toggle 开关
- 修改 `server/src/services/prompts.ts` — `buildSystemPrompt` 接收 `strictMode` 参数
- 修改 `server/src/routes/generate.ts` — 请求体中接收 `strictMode`，传给 `buildSystemPrompt`

**范围：** 小。约 40 行改动。

## P1 — 五、编辑体验

### 5.1 Markdown 工具栏

**问题：** 编辑区没有 Markdown 格式化工具栏。

**方案：** EditorPanel 顶部加浮动工具栏，6 个按钮：

| 按钮 | 操作 | 快捷键映射 |
|------|------|-----------|
| **B** 加粗 | 选中文本两侧加 `**` | Cmd+B |
| *H* 标题 | 行首加 `## ` | Cmd+H |
| • 列表 | 行首加 `- ` | Cmd+Shift+L |
| `<>` 代码块 | 插入 ```` ``` ```` | Cmd+Shift+C |
| — 分割线 | 插入 `---` | - |
| > 引用 | 行首加 `> ` | Cmd+Shift+Q |

实现方式：直接用 `document.execCommand` 或 textarea selection 操作。

**涉及文件：**
- 新建 `frontend/src/components/MarkdownToolbar.tsx` — 工具栏组件
- 修改 `frontend/src/components/EditorPanel.tsx` — 在 textarea 上方嵌入工具栏
- 修改 `frontend/src/components/EditWorkspace.tsx` — 注册工具栏的快捷键

**范围：** 小-中。约 80 行新组件。

## P2 — 二、用户漏斗

### 2.1 生成进度透明化

**问题：** 生成过程中 spinner 不透明，用户不知道"正在干什么"。

**方案：** 将当前 4 阶段伪进度改为更细粒度的 5 阶段：

```
🔍 分析仓库结构 (0-2s) → 🤖 AI 生成内容 (2-6s) → ✅ 格式优化 (6-8s) → 📋 质量检查 (8-10s) → ✨ 完成
```

每阶段显示具体文案，配合进度条颜色变化（当前已有类似实现，主要优化文案颗粒度和阶段数量）。

**涉及文件：**

**范围：** 极小。约 15 行改动。

## P3 — 七、增长闭环

### 3.1 README 底部标识

**问题：** 生成的 README 底部没有来源标识，无法给产品带来曝光。

**方案：** 在生成的 Markdown 末尾追加 HTML 注释（不影响渲染）：

```markdown
<!-- Generated by ReadMeCraft (https://readme-craft.com) -->
```

追加位置：后端 `buildUserPrompt` 返回结果的 content 末尾。确保用户手动编辑后不会丢失——在前端预览中也保留注释行。

**涉及文件：**
- 修改 `server/src/services/prompts.ts` — 在返回的 markdown 末尾追加注释
- 修改 `frontend/src/components/PreviewPanel.tsx` — 渲染时保留 HTML 注释（或不处理，不影响预览）

**范围：** 极小。2 行改动。

### 6.2 状态分层

**问题：** useReducer 已膨胀到 ~24 action 类型，接近拆分临界点。

**方案：** 按关注点拆分为 3 个 context：

| Context | 状态 | actions |
|---------|------|---------|
| `RepoContext` | repoUrl, repoInfo, repoLoading, repoError | FETCH_REPO_* |
| `EditorContext` | title, sections, activeSectionId, collapsedSections, history, historyIndex | SET_TITLE, UPDATE_SECTION, ADD/DELETE/MOVE_SECTION, UNDO/REDO, RESTORE_FROM_HISTORY |
| `UIContext` | selectedTemplate, generating, generateError, toast, toasts, showResultCard | SELECT_TEMPLATE, GENERATE_*, SHOW/DISMISS_TOAST, SHOW/HIDE_RESULT_CARD |

**涉及文件：**
- 新建 `frontend/src/context/RepoContext.tsx`
- 新建 `frontend/src/context/EditorContext.tsx`
- 新建 `frontend/src/context/UIContext.tsx`
- 修改 `frontend/src/context/AppContext.tsx` — 变为组合 3 个 context 并 re-export
- 修改所有消费 context 的组件 — 改为从对应 context 取值

**范围：** 大。约 200+ 行变更 + 10+ 组件需要改 import。**建议延后到其他功能稳定后再做。**

### 6.4 后端分析存储替换

**问题：** analytics.ts 用 NDJSON 纯文本存储，容器重启丢失、并发写入有问题。

**方案：** 替换为 SQLite（`better-sqlite3` 同步 API，无需异步 ORM）：

```sql
CREATE TABLE events (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  type TEXT NOT NULL,
  timestamp INTEGER NOT NULL,
  data TEXT
);
```

**涉及文件：**
- 修改 `server/src/services/analytics.ts` — NDJSON → SQLite
- 修改 `server/package.json` — 新增 `better-sqlite3` 依赖
- 删除 `server/data/events.ndjson`

**范围：** 中。约 100 行重构。
